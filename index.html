Computational Models in Cloud Computing



Cloud Computing is the provision of computing resources over the Internet. This is in the form of IaaS, PaaS and SaaS, provided on-demand. Typical cloud computing services include data storage, virtual servers, analytics, streaming and networking. This seemingly limitless stock of computing resources, reduces the global expense on hardware and increases the flexibility of choice in the vast amounts of hardware you can access. Hence, a cloud based infrastructure facilitates our requirements to solve real-world problems involving big data. Our research project focusses on some of the most popular cloud computational models developed to solve the different big data problems.

Prerequisites


Fault Tolerance

Fault tolerance is the ability of a system to recover nodes that have failed as the result of a fault. Fault tolerance can be achieved through many ways:

Checkpointing: In a fault-tolerant distributed system, this is the saving of data to stable storage at regular intervals to enable the recovery of lost data.

Lineage tracking: This is the tracking of dependencies of the datasets so that when a failure occurs, the dependencies can be retraced to recompute the lost data.

Ping test: Signals are sent to each node in the cluster and if they do not respond, they have failed.


Distributed Systems

A distributed system is a network of various software elements on several computers which share and divide computing resources so they can act as one integrated network to provide a single service to the user. When a distributed program is run on the distributed system, the job is subdivided into several tasks which are handed out to the various components of the network to execute.


Servers

A server which is a software or hardware device which takes responses from a client, performs the computation and returns the result.


Clusters

A cluster is a set of servers which are interconnected so that they can serve as one computing resource. A server within the cluster is referred to as a node. The nodes in the cluster are able to work individually or together to provide support for parallel applications.

Problem Classes


MapReduce, TensorFlow and Spark are three examples of frameworks that meet different demands of users. MapReduce and Spark are more often used in achieving results such as analytics, whereas Tensorflow is more likely to be used in improving software utility performance. TensorFlow and Spark both deal with iterative computations efficiently whereas MapReduce suffers high latency due to the disk I/O. TensorFlow is also sometimes used in conjunction with Spark or MapReduce to take advantage of their ability to deal with Big Data applications which is still superior to Distributed TensorFlow. Although Spark has the MLlib library for machine-learning, TensorFlow is far more specialised for this application. As demonstrated, we can see that some cloud computing frameworks are better suited to solve certain styles of problem more than others which means problems can be categorised by frameworks.

MapReduce


How are large datasets processed in a era where Big Data thrives and data is present everywhere? Which way can cloud computing help tackling these problems? MapReduce is a programming model and associated implementation invented by Google to deal with indexing the exploding volume of the content of the web by processing and generating Big Data sets with a parallel, distributed on a cluster. Usability, scalability and fault-tolerance are keys to its great success.

TensorFlow


Deep Learning has massively influenced the field of Computer Science, with these ideas giving rise to highly functional applications with ‘smart’ abilities. Google was making use of these techniques to power the algorithms behind its search, mail, photos and speech recognition tools. At the time, this was being done with its DistBelief framework, but as the need for machine learning grew, a more robust and efficient implementation was needed. TensorFlow arose from the need to generalise different machine learning implementations across Google to allow engineers to reuse the basic code base for future projects. After open-sourcing TF, Google released a distributed version of the framework alongside its Google Cloud Machine Learning infrastructure to train and serve TF models. This provided an ML framework which could be used by Google’s MapReduce, which struggled with highly iterative processes.

Apache Spark


Spark is a cluster computing framework, designed as a response to some of the deficiencies of the MapReduce model. For example, iterative computations and interactive applications are better supported by Spark than MapReduce, primarily due to Spark’s use of RDDs (resilient distributed datasets) which allow in-memory computations to be performed on clusters which increase the speed of computation. This is because the data can be accessed much faster from the RAM, than from the disk. In iterative algorithms which work on the same dataset multiple times, need to be able to read and update the data quickly; similarly, in interactive applications such as interactively querying a particular dataset also requires us to access memory repeatedly. The RDD data abstraction allows us to “persist” a particular set of data while we are working on it, so jobs are executed with less latency due to reading from the disk.
